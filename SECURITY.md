# Security

Do not open public issues for vulnerabilities or accidental exposure of credentials, private client data, buyer contact lists, profile data, or generated application artifacts.

Report security issues privately through GitHub's private vulnerability reporting when available, or contact the maintainers through the repository owner.

## Data Boundary

This package is intended to contain reusable harness code only. Consumer projects should contain private profile data, credentials, source lists, raw leads, reports, and application state.
